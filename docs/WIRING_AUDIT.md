# SignalOS Wiring Audit — Consolidated Report

## Headline Verdict

Across all three categories, the declared governance machinery is **substantially more decorative than the "fully wired & enforced, never advisory" promise claims**. Of the SignalOS-native runtime, only a thin spine is live: the harness step-* hooks fire (fail-open), the six G0–G5 gate agents dispatch as LLM prompts, and Python's `agent_loop._check_governance` genuinely denies tool calls for four rules (gate-gating, secret-block, trust-tier, test-first). Everything else is either **produced-then-ignored** (the Solution-Architect `arch_review` and the interface/data-model `design_decisions` contracts are signed and then never read by the code generator — the single most damaging gap), **advisory-only** (all PO/PE/QA/DevOps role-gating on the live manual sign path is bypassed via a hardcoded `--role PO`), **plugin-only** (session-start, pre-commit, pre-merge, pre-tool-use-guard fire only when SignalOS runs as a Claude/Cursor plugin, never from its own runtime), or a **pure no-op** (6 of 12 enforcement rules, plus five .md agents, are declared everywhere and read by nothing). Critically, the production IPC server passes `enforcement_provider=None`, so every rule mode the UI toggles is silently discarded — the agent loop runs on hardcoded static defaults and the Rust `EnforcementStore` the UI edits is never consulted.

## BIGGEST GAPS (prioritized)

1. **The Solution-Architect / plan contract never reaches code generation (produced-signed-then-ignored).** `arch_review` (delivery.py:460/466/474) and `design_decisions` (delivery.py:431/437/452) are built, validated, and signed, then dropped: `build_generation_packet` (generation.py:327) has no arch/plan parameter and prepare_generation (delivery.py:535-544) passes only the design SYSTEM, never the interface/data-model contract. The G2 plan agent's authority is decorative — the generator builds off intent/blueprint/design/acceptance and ignores the architecture it just signed. **This is the core "defined-but-dead" failure: the product signs an architecture contract it does not obey.** Fix at delivery.py:535 + generation.py:build_generation_packet.

2. **Six of twelve enforcement rules are pure no-ops.** plan-gating, audit-append, stack-contract, gate-compliance, zero-manual-regression, and mutation-threshold are declared in Rust ALL_RULES and Python RUNTIME_RULES but read by **no enforcement code path anywhere**. The UI lets a user toggle them; nothing consumes the toggle. This directly falsifies "fully wired & enforced, never advisory."

3. **UI rule modes are silently discarded in production.** `_build_agent_enforcement` returns `None` (signalos_ipc_server.py:497-505), so AgentLoop falls back to `StaticEnforcementProvider` all-strict defaults. The Rust `EnforcementStore` modes the UI edits via `set_rule_mode` are **never read by the Python policy** — even the four rules that DO enforce (gate-gating, secret-block, trust-tier, test-first) run on hardcoded defaults, not the user's configuration. Toggling any rule in the UI has zero effect on the agent loop.

4. **All role-gating on the live manual sign path is bypassed.** The UI → Rust `ipc::sign_gate` → sidecar → `signalos_ipc_server.sign_gate` path **hardcodes `--role PO`** (signalos_ipc_server.py:1364) and calls `sign_artifact` (no per-gate role check), never `sign.sign_gate` (the only role-enforcing function). The Rust `role_can_sign`/`check_role_for_gate` exist and are called, but only **advisorily** from JS (app-v2.js:376) — skippable, and `sign_gate` itself never consults them. A PO can manually sign G5 (a QA gate). Role enforcement is real only on the autonomous path, where the role is auto-derived and passes by construction — so it is never a user-facing authorization control.

5. **wave-freeze enforcement is now live.** freeze_wave/unfreeze_wave write the persisted frozen state, and both build entrypoints call filtered `build_precheck({ rules: ["wave-freeze"] })` before dispatch. A frozen wave now blocks chat delivery and approved-plan/retry wave dispatch before Build starts.

6. **Hook cluster #19 is now product-wired where it has product semantics.** pre-deploy is enforced by `signal-land-deploy` before deploy mutation, post-retro is enforced by the new `signal-post-retro` closure command, and step-pause receives `SIGNALOS_PLAN_STEP_JSON` from harness `_fire_hook`. `pre-session-compress` remains a reference shell hook because the disk-truth refusal invariant is enforced in-product by the Python mirror in `context.py`.

Of eleven markdown agents, only six (G0–G5) dispatch at all.
