## Engineering Discipline Guidance Pack

**Component:** Engineering Discipline guidance skills — `think-before-coding`,
`simplicity-first`, `surgical-changes`, and `goal-driven-execution`
(`core/execution/skills/*/SKILL.md`).

**Source repository:** https://github.com/multica-ai/andrej-karpathy-skills

**Relationship:** The four SKILL.md documents in this pack are original,
SignalOS-voiced rewrites. They adapt the *principles* expressed in the
MIT-licensed `multica-ai/andrej-karpathy-skills` repository, which in turn
derive from Andrej Karpathy's public observations on common LLM coding
pitfalls. No text from the source repository is reproduced verbatim; only the
underlying ideas are adapted, and SignalOS-specific framing (Belief/intent,
scope boundaries, acceptance criteria, and proof) is original to this project.
