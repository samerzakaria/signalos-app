# Amendments

`Canonical path: core/governance/Retro/AMENDMENTS.md · Append-only · Authored by: PO + PE · Signed by: PO + PE (+ Client if engagement model requires) · Cadence: triggered by Constitution §13 amendment path`

> The log of every change to SignalOS itself that a product has adopted or proposed back upstream. Three kinds of entries live here: **local amendments** (changes a product makes to its own instance), **Core distro amendments** (ratified changes to this repo's own release line), and **upstream proposals** (changes a product wants folded into the next SignalOS release).

---

## Local amendments

Changes to *this product's* Constitution / Gates / Validators that diverge from the SignalOS distro. Every local amendment is a forked rule — the product must re-affirm it every quarter, or it expires.

*(Empty at Gate 0 — products should prefer upstream proposals over local amendments unless there's a product-specific reason.)*

## Core distro amendments

Ratified amendments to the SignalOS Core distro itself. These are not temporary product forks; they are the release-history amendments for this repository.

**W1.1 hash anchor note.** The W1.1 amendment set anchored on `core/TRUST_TIER.md` because the v1.0.3 baseline did not yet ship a separate `core/CONSTITUTION.md` at the Core root. `absent → <sha256>` means the anchor file was introduced in that Wave.

**W1.2 hash anchor note.** W1.2 anchors on `cli/requirements.txt` because the distinguishing surface change of the Wave is the introduction of the first (and, by design, only) runtime third-party Python dependency in the entire W1.x series: `anthropic>=0.39,<1.0`. The anchor records the transition from the comment-only baseline to the pinned dependency.

**W1.3 hash anchor note.** W1.3 is two amendments, each with its own anchor. AMD-CORE-005 anchors on `cli/signalos_lib/context.py` — the rule-based compressor grew from a ~20-line stub placeholder to the full four-layer rule engine, so the file hash is the cleanest single record of the surface change. AMD-CORE-006 anchors on `cli/signalos_lib/registry.py` for the same reason: the stub placeholder became the full `install / verify / list / uninstall / publish` surface plus the cosign-verification shim. Neither amendment touches `cli/requirements.txt` — W1.3 adds no runtime Python dep (cosign is an external CLI binary, shelled to only at install and publish time, and is guarded by the `SIGNALOS_REGISTRY_TEST=1` mock in every proof scenario).
