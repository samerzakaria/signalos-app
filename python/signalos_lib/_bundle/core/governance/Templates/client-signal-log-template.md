# Client Signal Log — {Product Name}

`Canonical path: core/governance/Governance/CLIENT-SIGNAL-LOG.md · Authored by: PO · Updated: every client reaction, positive or negative · Format: verbatim, append-only`

> Every meaningful client reaction to a shipped change is logged here **verbatim**. No paraphrasing. No editorialising. If the client said "that's confusing", the entry says that's confusing — not "client had some UX concerns." The Wave Review + Debrief reads this log to decide Keep / Kill / Iterate.

---

## How to use this file

- **After every client touchpoint** (demo, check-in, email reply, support ticket referencing a shipped change), append an entry.
- **Quote the client verbatim** in their own words, in the language they used.
- **Never edit entries.** If a quote needs context, add a new entry with a pointer.
- **Triage to Backlog** if a reaction contains a new CR — log both here and in `core/governance/Templates/cr-log.md`.
